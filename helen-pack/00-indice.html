Paquete Helen · índice y marco editorial

Este documento abre el paquete. Explica qué hay en cada uno de los 5 docs (más una skill ejecutable), bajo qué marco editorial fueron construidos, y en qué orden conviene leerlos según el tipo de necesidad. Tres minutos de lectura.

5 docs HTML + 1 skill · standalone · mobile-friendly · 18 may 2026

0 · Qué es este paquete

Un set de cinco documentos HTML autocontenidos más una skill ejecutable de IA, armados como soporte para el lanzamiento del disco nuevo de Helen Olhausen (Humana, agosto 2026) y para la dirección creativa del proyecto.

Cada doc es un archivo único que abre con doble click en cualquier navegador (Chrome, Safari, Firefox), sin conexión a internet, en desktop o mobile. Se pueden compartir por mail, WhatsApp o Drive como archivos sueltos.

La skill es un archivo de texto (SKILL.md) que se carga en Claude (web o local) para tener un asistente que chequea decisiones de dirección creativa contra patrones validados. Viene con su archivo de fuentes auditables.

1 · Marco editorial · cómo se armó este paquete

El paquete está construido bajo cuatro grupos de directrices explícitas provistas por Emi (Emiliano Mettini) durante la construcción del paquete. Quedan documentadas acá para que la lectura sea verificable.

Contenido y framing

  • Voz + síntesis como eje del disco, no modular industrial
  • El disco se llama Humana, release agosto 2026
  • Asume que el armado del disco (master, tapa, distribución) está siendo manejado por Helen — el paquete cubre solo difusión y soporte estratégico

Estructura del paquete

  • 5 docs HTML standalone + 1 skill ejecutable
  • El deck de difusión tiene tres bloques visualmente diferenciados (pre-lanzamiento, durante el lanzamiento, post-lanzamiento)
  • Cada doc lleva bibliografía completa + fuentes auditables inline
  • HTML standalone: abre con doble click, sin servidor
  • Mobile-friendly + print-friendly
  • Tono profesional pero cercano

Rigor

  • Sustento auditable en todo lo que se afirma — validación de fuentes con URL inline
  • Honestidad sobre probabilidades — escenarios con % explícito cuando aplica
  • Caveats explícitos cuando falta data
  • Correcciones declaradas (Marie Davidson ≠ Pelada, Elsa Hanneke ≠ DIA, Querida no es de Eslovenia, DÍA es 2025 no 2020)
  • Casos de estudio con estructura repetida: trayectoria + pivot + ecosistema + equipo + lessons
  • La skill funciona como tablero de chequeo, no como dictamen creativo

Lo que NO es scope del paquete

  • Independencia económica como norte de carrera vital → tratada en doc 03 como marco estratégico de fondo, no como roadmap
  • Composición musical, lyrics, mezcla, mastering → no aborda
  • Estrategia de marketing pagado → mención solo, no profundización

2 · Mapa de los documentos

Cinco docs HTML + una skill. Cada uno tiene un rol específico.

01 · deck

Deck de difusión del lanzamiento

helen-pack/01-deck-difusion.html

Plan de ejecución para el release de agosto. Tres bloques diferenciados: pre-lanzamiento (decisiones de release + pitches críticos a prensa + reactivación del catálogo), durante el lanzamiento (drop coordinado, ventana de reviews), post-lanzamiento (mantener calor + cola larga). Plus: research del contexto (artistas similares, prensa por tier, playlists, comunidades, KPIs con probabilidades, riesgos, bridge messaging). Cierra con una sección sobre la difusión satelital paralela.

Cuándo abrirlo: planificando acciones del release, mirando contactos de prensa, buscando KPIs realistas, o revisando bridge messaging.
02 · casos

Casos de estudio · 5 artistas + sus pivots

helen-pack/02-casos-estudio.html

Análisis profundo de cinco artistas que recorrieron caminos adyacentes: Juana Aguirre (caso central), Ela Minus, Marie Davidson, Lila Tirando a Violeta, Lucrecia Dalt. Para cada una: trayectoria, el pivot que cambió la escala, ecosistema económico actual, equipo, lessons aplicables. Al final, cinco patrones transversales con cobertura honesta de evidencia.

Cuándo abrirlo: cuando aparezca la pregunta "¿qué hicieron las que llegaron?", para mirar trayectorias comparables sin idealizar.
03 · economía

Camino a la independencia económica indie

helen-pack/03-independencia.html

Síntesis macro sobre cómo se construye un ecosistema económico sostenible para un artista indie. Mapea las nueve fuentes de ingreso reales, cuánto pesa cada una típicamente, qué hicieron los cinco casos para articularlas, y cinco decisiones estratégicas estructurales (sello, base geográfica, frecuencia de releases, tipo de show, transición del trabajo paralelo). Marco de pensamiento, no roadmap.

Cuándo abrirlo: cuando aparezca la pregunta "¿cómo se sostiene esto a 5-10 años?", para tener marco macro sin confundirlo con plan de acción del release.
04 · directoras

Directoras creativas · método y 7 patrones

helen-pack/04-directoras.html

Análisis del método de cinco referentes que dirigieron la identidad visual de proyectos musicales con autoría visible: Rui Vieira, Ana Domínguez, Querida, Elsa Hanneke, Studio Albert Romagosa + el modelo Lemonade. Para cada una: background, criterio indie aplicado, método de conexión visual ↔ sonoro, caso puntual. Al final, los 7 patrones cruzados que son la base de la skill creative-direction.

Cuándo abrirlo: para reuniones con audiovisualistas, decisiones sobre tapa o videoclip, briefear colaboradoras visuales, o pensar el show en vivo.
05 · cuentas advisory

Cuentas, plataformas y presencia digital · advisory completo

helen-pack/05-cuentas-advisory.html

Auditoría plataforma por plataforma de la presencia digital actual de Helen (Spotify 1.564 followers, Last.fm 374 listeners, etc.) + lo que falta (Spotify for Artists sin reclamar, MusicBrainz/Wikidata vacíos, handles X/Bluesky/TikTok libres, helenolhausen.com libre). Estrategia de timing en dos olas: Wave 1 (infraestructura + identidad, ahora) y Wave 2 (activación creativa, con el disco nuevo). Doc cubre tier por tier cada plataforma, sección dedicada a si vale la pena escribir/blog/newsletter + 10 ideas de essays.

Cuándo abrirlo: ahora — antes de empezar a operar la presencia digital. La auditoría es de mayo 2026.
skill

Skill creative-direction (ejecutable en Claude)

.claude/skills/creative-direction/ — SKILL.md + SOURCES.md

Una skill que se carga en Claude (web o local) y le agrega comportamiento de chequeo creativo. Cuando se le consulta una decisión visual sobre el disco, responde validando contra los 7 patrones del doc 04 (variable única, materia prima compartida, decisión material, doble rol artista/traductora, inversión de tropos, equipo estable, lectura antes que Pinterest). Tablero de chequeo, no dictamen creativo. Viene con SOURCES.md que documenta cada afirmación con URL.

Cuándo usarla: cuando hay una decisión visual sobre la mesa (cover, video, performance) y se busca una segunda lectura estructurada. Se invoca con frases como "validá esta decisión", "qué haría Querida con esto", "estoy armando el mood board".

3 · Orden de lectura sugerido

No hay que leerlos todos seguidos. Cada uno responde a una necesidad distinta — abrirlos cuando esa necesidad aparezca. Pero si querés un primer recorrido completo, este es el orden que tiene más sentido:

  1. 1. Este doc (00) — marco editorial y mapa. Ya estás acá.
  2. 2. Deck de difusión (01) — porque tiene la temporalidad inmediata: agosto está cerca y muchas decisiones de pre-lanzamiento tienen ventanas cerrándose.
  3. 3. Directoras creativas (04) — porque mañana hay reuniones con audiovisualistas y los 7 patrones sirven YA, antes de cerrar decisiones visuales.
  4. 4. Skill creative-direction — cargarla en Claude para usarla cuando aparezca cualquier decisión creativa que necesite segunda lectura.
  5. 5. Casos de estudio (02) — cuando haya energía para mirar trayectorias completas. Más denso, más largo, no urgente.
  6. 6. Independencia económica (03) — cuando aparezca la pregunta "¿cómo sostengo esto a largo plazo?". Marco macro, no para el día a día del release.
Si tenés poco tiempo: los docs 01 y 04 son los más operativos. El primero da el plan del release inmediato; el segundo da los patrones para las decisiones visuales que están en juego mañana. Los demás profundizan contexto.

4 · Rigor y solidez de la información

No todo lo que está en el paquete tiene el mismo nivel de confianza. Algunas afirmaciones son hechos verificables con un click; otras son inferencias razonadas marcadas como tales. La tabla que sigue mapea cada tipo de afirmación con su nivel de confianza y por qué.

Las afirmaciones de baja confianza están declaradas como inferencia o caveat dentro de cada doc — no se presentan como certezas. Las de alta confianza tienen URL inline verificable en su lugar.

Tipo de afirmación Confianza Por qué
URLs citadas + facts directos (fechas, sellos, releases) alta Verificables con un click. Cada claim factual tiene fuente.
Quotes textuales de entrevistas alta Citables literalmente con fuente. Reproducibles.
Trayectorias y pivots de cada artista media-alta Cruzadas entre múltiples fuentes. Interpretación del peso relativo es nuestra.
Ecosistema económico (porcentajes por fuente) baja-media Inferencias razonadas sobre catálogo público, no contabilidad verificada. Declarado como tal en cada caso.
Rangos de fees / stipends de industria media Datos de industria 2024-2025 (Variety, Hollywood Reporter, Soundcharts). Pueden estar desactualizados o variar por mercado.
Patrones transversales (5 de casos + 7 de directoras) media Interpretaciones derivadas del research. No validadas por terceros del sector. Cobertura declarada (X/5).
Probabilidades de escenarios de growth (65% / 25% / 3-5%) baja Estimaciones basadas en casos comparables. No son estadística — son rangos para medir si la campaña está rindiendo.
Cómo se puede aumentar la solidez: validación externa (revisión por gente del sector — managers, periodistas, productores), tracking temporal contra escenarios planteados, y cruce con feedback de Helen sobre lo que conoce de primera mano. Esa capa se puede armar después si interesa.
