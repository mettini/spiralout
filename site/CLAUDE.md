# site/ — the public website

The Spiral Out website. Deployed to **Cloudflare Pages** as project
`spiralout` at **https://spiralout.space**.

## Layout

```
site/spiralout/
├── index.html              ← /        (umbrella brand page)
├── aem/
│   └── index.html          ← /aem/    (ÆM artist + Heliopause album landing)
│   └── share/              ← per-network share images for /aem
├── share/                  ← per-network share images for the home
├── og-home.{svg,jpg}       ← primary OG (1200x630) for /
├── aem/og-image-wide.{svg,jpg}  ← primary OG (1200x630) for /aem
├── aem/og-image.jpg        ← raw album cover (3000x3000) for music.album OG
├── favicon.* / site.webmanifest / robots.txt / sitemap.xml
└── spirals/                ← spiral playground page (internal)
```

Two surfaces, one site:

- **`/`** — minimal umbrella page. The dotted spiral logo, "spiral out"
  wordmark, a phrase, and one link out to `/aem/`. No build step. Pure HTML
  + inline CSS. Animated SVG spiral.
- **`/aem/`** — artist + album landing. Hero is the Heliopause cover.
  Bandcamp link is live; other platforms are placeholders (disabled
  visually, will go live as the album ships to Spotify / Apple Music / etc).

## Stack

- **Plain HTML/CSS** — no framework, no build step, no bundler. Inline
  `<style>` blocks per page. The reason: the site is small, fast-loading,
  and survives renderer quirks (preview crawlers, embed cards).
- **No JS** on the public pages (the home has an `<svg>` spiral animated
  via CSS keyframes — that's it).
- **Fonts**: system `'Courier New', ui-monospace, monospace`. Don't add web
  font requests. The typewriter aesthetic IS the brand. Match it.
- **Color tokens** (mirrored in `:root` CSS vars on every page):
  - `--bg: #0a0a0c`
  - `--fg: #cfcfd2`
  - `--muted: #555560`
  - `--accent: #8a7a90` (the spiral / hover color)
- **Aspect of the spiral**: dotted, growing radii, ~80 circles. The exact
  data lives in `index.html` AND in `scripts/generate_share_images.py`
  (`SPIRAL_DOTS`). If you change one, sync the other.

## Share images (Open Graph / social previews)

All share images are generated by `scripts/generate_share_images.py`. Don't
hand-edit the JPGs — re-run `task site:share` instead. The generator emits:

| File                              | Size       | Target networks                                                       |
|---|---|---|
| `og-home.jpg`                     | 1200x630   | Default OG: FB / LinkedIn / Discord / WhatsApp / X / iMessage / Threads / Bluesky / Mastodon / Telegram |
| `share/home-square.jpg`           | 1080x1080  | Instagram feed square (manual upload)                                 |
| `share/home-portrait.jpg`         | 1080x1350  | Instagram feed portrait 4:5 (highest reach)                           |
| `share/home-story.jpg`            | 1080x1920  | Instagram / TikTok story (top + bottom 250px = UI safe zone)          |
| `share/home-pinterest.jpg`        | 1000x1500  | Pinterest pin 2:3                                                     |
| `aem/og-image-wide.jpg`           | 1200x630   | Default OG for /aem                                                   |
| `aem/og-image.jpg`                | 3000x3000  | `music.album` OG (raw album art) + Bandcamp ingestion source          |
| `aem/share/aem-{square,portrait,story,pinterest}.jpg` | per-network | manual upload                       |

Compression target: **keep every `og-*.jpg` under 300 KB**. WhatsApp and
Bluesky silently drop previews above ~1MB. Current sizes are well under.

The home OG design respects the home page typography (Courier New, weight
300, wide letter-spacing, lowercase) — if you change the home `<h1>`, update
the OG too.

The `/aem` share images treat **the album cover as the hero** (artist promo
intent). The cover already contains ÆM / HELIOPAUSE / TRANSMISSION 01 +
tracklist baked in, so the wide / portrait / story variants only add a
minimal title block beside or below the cover.

## Meta tags

Each page has the full OG + Twitter + Schema.org block in its `<head>`.
When you edit a page, **check all three locations** stay in sync:

- `<title>` / `<meta name="description">`
- `og:title` / `og:description` / `og:image` / `og:image:alt`
- `twitter:title` / `twitter:description` / `twitter:image`
- The Schema.org `<script type="application/ld+json">` block

Don't introduce a templating engine to dedupe these — the site is two
pages. Keeping them inline is the right tradeoff.

## Deploying

Cloudflare Pages, project name `spiralout`, branch `main`. From repo root:

```bash
task site:dev      # local preview on :8788 (wrangler pages dev)
task site:deploy   # wrangler pages deploy site/spiralout
task site:list     # last deployments
```

First time setup: `task site:install` (installs wrangler) and
`task site:login` (OAuth in browser).

Note: `--commit-dirty=true` is intentional — there's no Git remote tied to
the Cloudflare project, deploys are by content.

## When you change things

- **Edit a page** → preview locally with `task site:dev` before deploy.
- **Edit the spiral geometry** → regenerate OGs with `task site:share`.
- **Add a new release** → create `/transmission-NN/` or a sister to `/aem/`
  with its own meta block + share images. Don't reuse the ÆM template
  blindly — the brand convention is per-release artist pages.
- **Change brand color** → update `:root` vars on every page AND the
  palette constants at the top of `scripts/generate_share_images.py`.

## Things NOT to do

- Don't add a build step (Vite, Next, Astro, anything). The constraint of
  "two pages, inline CSS, no JS" is intentional.
- Don't add web fonts. Courier New everywhere.
- Don't replace the SVG spiral with an image — it needs to be vector for
  the favicon, OG images, and the animated home page.
- Don't bump deps in `wrangler` without checking — Cloudflare Pages is
  picky about wrangler versions.
