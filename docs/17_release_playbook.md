# 17 — Release marketing playbook

Plan de difusión para sacar un release de ambient/experimental indie. Te
sirve como playbook reusable para **Heliopause (ÆM)** y para
**el disco de Helen en agosto**. Ver
[`docs/18_helen_olhausen_campaign.md`](18_helen_olhausen_campaign.md)
para el ajuste específico de Helen.

Tres premisas:

1. **Ambient indie no se vende con shouting**. Construís pertenencia: te
   metés en una conversación / scene, no la interrumpís.
2. **El descubrimiento es por adyacencia**. Fans encuentran nuevos artistas
   vía playlists, blogs, "fans también escuchan…". Tu trabajo es estar en
   los nodos adyacentes correctos. Acá entra
   [`docs/19_resonance_integration.md`](19_resonance_integration.md).
3. **Distribución barata + tiempo > plata**. El presupuesto va para Spotify
   Marquee / Meta Ads sólo cuando ya tenés tracción orgánica.

## Pre-release timeline

Asumiendo release date = **R**. Las semanas se cuentan hacia atrás (T-N).

### T-90 (3 meses antes)

**Producción**:
- [ ] Master finalizado, LUFS-streaming correcto (-14 LUFS aprox, ver
      `aem-release` skill para targets 2026)
- [ ] Tapa final aprobada (3000x3000 sRGB JPG mínimo)
- [ ] Tracklist + duraciones congeladas
- [ ] Letras / liner notes / credits congelados

**Plataformas**:
- [ ] Decidir distribuidor: **CD Baby vs DistroKid**.
  - CD Baby: pago único por release, royalties pass-through, YouTube
    Official Artist Channel claim incluido, soporte de cobertura editorial.
    Mejor para singles esporádicos.
  - DistroKid: subscripción anual unlimited, sale más barato si tirás
    múltiples releases por año. Estándar de la industria indie hoy.
  - **Para ÆM (un álbum al año, quizás 2)**: DistroKid es más rentable.
  - **Para Helen (un álbum cada varios años)**: CD Baby pago único es más
    simple y resuelve el YouTube claim mejor.
- [ ] Crear cuenta en el distribuidor elegido

**Identidad**:
- [ ] Todas las cuentas de la
      [`checklist 15`](15_brand_accounts.md) sacadas
- [ ] Sitio + landing del álbum funcionando (ya)
- [ ] Press kit armado (ver sección X)

### T-60 (2 meses antes)

**Distribución**:
- [ ] Subir release al distribuidor con fecha = R
- [ ] **Asegurar Spotify pitch eligibility** — necesitás distribuir al menos
      7 días antes pero idealmente 30+ para que llegues al "Pitch to
      Editorial" de Spotify for Artists. Sin esos 7 días no pitcheás.

**Press**:
- [ ] Pitchear a blogs (lista en sección PR abajo)
- [ ] Pitchear a Bandcamp Daily — tienen formulario propio
- [ ] Enviar a curadores de playlist (SubmitHub paid tier vale la pena para
      ambient, evita rebote ruidoso)

**Social**:
- [ ] Primer post de "coming soon" con teaser visual de la tapa
- [ ] Cover reveal completo cuando llegues a T-45/T-30

### T-30

**Pre-save / pre-order**:
- [ ] Bandcamp pre-order live (con bonus track, alta-res download, etc.)
- [ ] Pre-save de Spotify via [Linkfire](https://linkfire.com) o
      [ToneDen](https://toneden.io) (los pre-saves no son automáticos en
      Spotify; necesitás un servicio third-party)
- [ ] Spotify for Artists → **"Pitch to Editorial"** (esta es la
      ventana). Mínimo 7 días, idealmente 14+. Llenar todos los campos:
      mood, genre, instruments, lyrics theme, story behind the song.

**Contenido**:
- [ ] **Single 1** drops (un track del album, lyric video / visualizer
      simple en YouTube, posted everywhere). Para ambient: visualizers son
      el "video" — generative / waveform / 1 imagen quieta de la tapa
      con audio.

### T-14

- [ ] **Single 2** drops (otro track o un remix / collaboration)
- [ ] Apple Music for Artists pitch (limit: 1 song por release, así que
      eligen su track favorito)
- [ ] Indie playlist pitching round 2

### T-7

- [ ] **Newsletter de lanzamiento** (Buttondown / Substack — gratis hasta
      X subs, te recomiendo Buttondown que es indie-friendly)
- [ ] Stories countdown en IG
- [ ] Recordatorio en Bluesky / X

### Release day (T-0)

- [ ] Drop post coordinated en todas las redes (mismo timing,
      diferentes copy: IG con visual, Bluesky con texto + link, etc)
- [ ] YouTube full album upload (si lo permitís) + lyric videos / visualizers
- [ ] Bandcamp release day push (suelen tener "new arrivals" listing)
- [ ] Email newsletter blast
- [ ] Submit a [Album of the Year](https://www.albumoftheyear.org) (community-driven)
- [ ] Submit a [RateYourMusic](https://rateyourmusic.com) (vos o un fan)

### T+7

- [ ] Métricas pull desde Spotify for Artists / Apple Music for Artists /
      Bandcamp dashboards
- [ ] Si tracción orgánica es buena → considerar **Spotify Marquee**
      (campaña dentro de Spotify, ~$300-500 USD, hace push notification
      a usuarios que escucharon artistas similares)
- [ ] Si Bandcamp generó pre-orders sólidos → activar **Bandcamp Boost**
      (ads internos a $$, similar idea)

### T+30 (mantener calor)

- [ ] Behind-the-scenes content: making-of un track, foto del setup,
      proceso (los Python scripts SON contenido para fans de música tech)
- [ ] Posibles colaboraciones / remixes
- [ ] Live stream / DJ set ambient (Twitch / YouTube)
- [ ] Aplicar a podcasts de ambient (Echoes, Hearts of Space, Dark Ambient
      Garden Podcast)

### T+90

- [ ] Evaluar resultados: streams, listeners, country split, pre-saves,
      Bandcamp sales, social growth
- [ ] Diseñar el próximo release (transmission 02) con learnings
- [ ] Cosecha de press para usar en pitches futuros (quote sheet)

## PR — quién pitchear

### Tier 1 — blogs de ambient/experimental

Listar con URL + contacto pitch. Mantener spreadsheet aparte con tracking
"sent/replied/posted".

- [Fluid Radio](https://www.fluid-radio.co.uk) — UK, ambient/experimental
- [Headphone Commute](https://www.headphonecommute.com) — top tier para
  ambient
- [Stationary Travels](https://stationarytravels.wordpress.com) — ambient
  blog activo
- [Ambient Music Guide](https://www.ambientmusicguide.com) — directorio +
  reviews
- [A Closer Listen](https://acloserlisten.com) — instrumental/post-rock/ambient
- [The Quietus](https://thequietus.com) — más generalista pero cubre ambient
- [Tiny Mix Tapes](https://www.tinymixtapes.com) — experimental, cuesta
  entrar pero exposure grande

### Tier 2 — playlist curators

- **Spotify editorial** (vía Spotify for Artists, no contacto directo)
- **Independent curators** en Spotify (buscar playlists "ambient",
  "dark ambient", "deep space ambient" con 1k-10k followers — sus owners
  suelen estar en SubmitHub o tener formularios)
- **Bandcamp guests curators** (Bandcamp Daily lists)
- **Apple Music editorial** (vía Apple Music for Artists)

### Tier 3 — podcasts / radio

- [Hearts of Space](https://www.hos.com) — ambient radio show
- [Echoes Radio](https://www.echoes.org) — John Diliberto
- [Soma FM](https://somafm.com) — internet radio, varios channels ambient
- Local stations / Buenos Aires (Radio Nacional Clásica, etc — para Helen)

## Presupuesto de marketing (orientativo, USD)

Para un release indie ambient que recién arranca:

| Concepto | Costo | Cuándo |
|---|---|---|
| Distribución (CD Baby o DistroKid) | $9-23 por release o $20/año unlimited | T-90 |
| SubmitHub credits (paid tier) | $30-80 por release | T-60 |
| Linkfire / ToneDen pre-save | $0-15/mes | T-30 |
| Buttondown newsletter | $0 hasta 100 subs, $9/mes después | ongoing |
| Spotify Marquee (opcional) | $250-500 | T+7 si tracción ok |
| Meta Ads (IG/FB) | $50-200 budget piloto | T+0 a T+30 |
| **Total mínimo viable** | **~$50-100** | — |
| **Total con push pagado** | **~$400-800** | — |

**Regla**: nunca gastes en Meta Ads si no tenés mínimo 50 saves orgánicos
en pre-release. Los ads amplifican lo que ya funciona — no fabrican
tracción de la nada.

## Press kit (sección X)

Bandcamp tiene `/press` o mandar PDF. Estructura:

1. **Bio corta** (50 palabras) — para que copien y peguen
2. **Bio larga** (150-200 palabras) — para reviews
3. **3 high-res photos** — 1 cuadrada para grid, 1 horizontal para banner,
   1 vertical para story
4. **Cover art** en 3000x3000
5. **One-sheet PDF** con tracklist, fecha de release, label, distribution,
   contact, links
6. **Pre-release stream link** privado (Bandcamp tiene esta opción nativa)
7. **Quotes** de press anterior (si hay)

Mantener en `transmissions/<TX>/release/press_kit/` (gitignored).
