**Key findings**:
- Drone ambient = stripped to the marrow. Sustain en vez de rhythm. Presencia
  en vez de movimiento.
- Drone establece tonal center = anchor, "gravity that pulls everything
  together".
- Drones = filling space underneath active layers + setting mood + frequency
  root + timbral theme para countering otros sounds.
- Synth analog/soft. Saw rich para distortion; sine pure para sub bass.
  Layering distintos waveforms = complejidad.
- Add one layer at a time. New texture cada 30s = pace fino.
- Cada nueva capa cambia la perceived space o color.
- Automation: filter moves + reverb sends → track feels alive.
- Silencio es musical → tensión. Drop layers sin warning, traerlas con
  timbre distinto.
- Field recordings: fluorescent light buzz, washing machine, coffee grinder
  pitched down → spaceship funeral sounds.
- **Anti-pattern**: thin low end por cortar lows de todo. Mantener UNA drone
  baja y carve space para ella.

- Layering: combinar múltiples sound sources (tonal riser + filtered noise +
  reversed reverb tail). Cada elemento = una dimensión.
- Variar timing offsets, reverb tails, filter shapes → cada transición fresh.

  oscillators (NO sample replay, sino recreación). Aplicado a la voz de
  Molly Rogers (violinista) y Loire Cotler.

## William Basinski / Disintegration Loops (degradación de loops)

- [The Disintegration Loops — Wikipedia](https://en.wikipedia.org/wiki/The_Disintegration_Loops)
- [Memory, Decay, Activism (Eli Stine)](https://elistine.com/blog/memory-decay-activism-william-basinskis-the)
- [Disintegration Loops measures the weight of time (Treble)](https://www.treblezine.com/william-basinski-disintegration-loops-hall-of-fame-review/)

- [Disintegration and repetition (Academia paper)](https://www.academia.edu/40893603/Disintegration_and_repetition_an_analysis_based_on_William_Basinski)

**Key findings**:
- Tape loops antiguos (años 80, easy listening) recapturados en CD digital.
- En cada repetición la cinta DEGRADA físicamente — iron oxide se
  descompone, gaps de silencio aparecen.
- Resultado: melodic fragments accrue percussive artifacts (snaps, dust
  particles) → emergent textures. Real-time sonic erosion.
- Aplicable digitalmente: simular degradación con
  - amp envelope que decae loop tras loop
  - filter (LPF) que cierra progresivamente
  - dropouts (silencios random) cuya densidad crece con el tiempo
  - aumento gradual de noise/crackle overlay

## Reverse reverb (pre-event swell)

- [Reverse Reverb: 4 Ways (LANDR)](https://blog.landr.com/reverse-reverb/)
- [Reverse Reverb: What it is (Splice)](https://splice.com/blog/what-is-reverse-reverb/)
- [Reverse reverb effect (Native Instruments)](https://blog.native-instruments.com/reverse-reverb-effect/)
- [Set Up Pre Reverse Reverb (Production Expert)](https://www.production-expert.com/home-page/2016/10/5/tutorial-how-to-set-up-pre-reverse-reverb-delay-swell-effects-before-audio-in-pro-tools)
- [Instant Reverse Reverb Effect (Mastering.com)](https://mastering.com/reverse-reverb/)

**Key findings**:
- Reverse reverb = el tail del reverb se reproduce HACIA ATRÁS, llevando
  desde silencio al evento (en lugar del fade out clásico).
- Crea swell otherworldly, supernatural build-up.
- DAW workflow:
  1. Tomar el dry signal del evento
  2. Copiar a otro track + aplicar reverb generosa
  3. Reverse el track procesado
  4. Posicionar antes del evento original → cuando termina el reverse
     reverb cae el evento dry.
- En el framework: para una nota X, hacer
  `fade(reverb(melody([(X, dur)]))[::-1], fi=dur*0.6, fo=dur*0.05)` y
  ponerlo ANTES de la nota original.

## Tape saturation / lo-fi warmth

- [Tape Saturation: A Beginner's Guide (Producertech)](https://www.producertech.com/blog/tape-saturation-a-beginner-s-guide)
- [Tape Saturation: Make Digital Sound Warm (Baby Audio)](https://babyaud.io/blog/what-is-tape-saturation)
- [Analogue Warmth (Sound on Sound)](https://www.soundonsound.com/techniques/analogue-warmth)
- [Tape Saturation Tools for Neoclassical/Ambient (VI-CONTROL)](https://vi-control.net/community/threads/best-tape-saturation-simulation-tools-for-neoclassical-ambient-music-j%C3%B3hannsson-arnalds-sakamoto-inspired.162698/)

- [Vintage Warmth in Digital Productions (Karanyi Sounds)](https://karanyisounds.com/blogs/production-tips/bringing-vintage-warmth-to-digital-productions-how-to-achieve-an-authentic-analog-feel)

**Key findings**:
- Tape saturation: subtle distortion + harmonic enhancement + dynamic
  compression. "Soft clip" musical.
- Suaviza transients, redondea low end, agrega harmónicos pares (warm).
- Lower tape speeds (7.5 IPS) = más warmth y bass; higher (15-30) = cleaner.
- Aplicado al mix bus: glue cohesivo. Aplicado por track: cumulative warmth.
- Cómo aproximarlo en numpy: `tanh(audio * drive) / tanh(drive)` con drive
  bajo (1.5-2.0) = soft saturation. Diferente del `distort` actual (que es
  saturation fuerte 2.0-5.0 para wall of sound).
- Para "interferencia radio" (que es lo que le falta al voyager_vestige):
  combinar tanh saturation + noise overlay + LPF dinámico.

## Mastering ambient — LUFS streaming 2026

- [Loudness Mastering Streaming Platforms 2026 (Sean Kim)](https://blog.imseankim.com/loudness-mastering-lufs-streaming-platforms-spotify-apple-music-2026/)

- [Loudness normalization on Spotify (Spotify Artists)](https://support.spotify.com/us/artists/article/loudness-normalization/)
- [How to master for streaming (iZotope)](https://www.izotope.com/en/learn/mastering-for-streaming-platforms)
- [Spotify LUFS 2026 (Soundcamps)](https://soundcamps.com/blog/spotify-lufs/)
- [LUFS Standards (Genesis Mix Lab)](https://genesismixlab.com/blog/understanding-lufs-loudness-standards/)
- [Streaming LUFS Tested (Teknup)](https://www.teknup.com/streaming-lufs-tested-the-real-truth-about-loudness/)

**Key findings**:
- Sweet spot universal streaming 2026: **-14 LUFS integrated, -2 dBTP**.
  Funciona para Spotify/YouTube/Tidal/Amazon. Apple Music ideal -16 LUFS.
- Para AMBIENT específicamente: **mejor sonar más quieto y dinámico**
  (-14 a -16 LUFS, hasta -18). Heavy limiting destruye el género — la
  contraste entre silencio y swells es lo que hace inmersivo.
- Mantener al menos **8-10 dB de dynamic range** entre las partes más
  quietas y los peaks.
- Streaming SIEMPRE normaliza. No hace falta apuntar a -8 LUFS para
  "sonar fuerte" — sale aplastado al mismo volumen percibido.
- Single master strategy: NO subir versión Spotify y versión Apple
  separadas. Un master bien hecho funciona en todas.

## Metadata audio (Python: mutagen)

- [Mutagen Library (Aspose Document Processing)](https://products.documentprocessing.com/metadata/python/mutagen/)
- [music-tag (PyPI)](https://pypi.org/project/music-tag/)
- [pytaglib (PyPI)](https://pypi.org/project/pytaglib/)
- [tinytag (GitHub)](https://github.com/tinytag/tinytag)
- [Audio Metadata Schemas y File Formats (htcUtils)](https://htcutils.cloud/blog/complete-guide-to-audio-metadata-schemas-tags-and-file-formats)
- [Python Audio Tools — Metadata Formats](https://audiotools.sourceforge.net/programming/metadata.html)

**Key findings**:
- **mutagen** es la librería estándar Python para metadata audio. Soporta
  ID3v1, ID3v2, Vorbis Comments (FLAC/Ogg), APEv2, MP4 (m4a). Lectura
  + escritura + artwork embebido.
- **music-tag** = wrapper de mutagen con API uniforme cross-format.
- **pytaglib** = bindings de TagLib, también soporta artwork.
- **CRÍTICO**: WAV NO soporta artwork embebido. Para artwork tenés que
  usar FLAC/MP3/AAC.
- FLAC usa Vorbis Comments (key-value pairs) + Picture block para artwork.
- MP3 usa ID3v2 frames: TIT2 (title), TPE1 (artist), TALB (album), TRCK
  (track number), APIC (artwork).
- Tags estándar: TITLE, ARTIST, ALBUM, ALBUMARTIST, DATE, TRACKNUMBER,
  TRACKTOTAL, GENRE, COMPOSER, COPYRIGHT, ISRC.

- [Spotify Provider Directory](https://artists.spotify.com/en/providers)

**Key findings**:
- **DistroKid** $24.99/año unlimited tracks, 0% comisión. Ideal para
  bedroom producers que sacan EPs/singles seguido. Rápido a TikTok.
- **CD Baby** $9.99/single $14.99/álbum one-time, 9% comisión, hosting
  permanente. Ideal para catálogo permanente con baja frecuencia.
- **TuneCore** $24.99/año Rising / $54.99/año Pro. Publishing admin
  incluido en Pro. Bueno para covers o si querés royalties de
  publishing administrados.
- **Bandcamp directo**: gratis, 15% comisión primer pago / 10% después.
  Drop directo a fans, sin agregador. Recomendado para primer release.
- ISRC codes: dejar que el agregador los genere (gratis). Solo registrar
  prefijo propio en IFPI si vas a sacar mucho catálogo.
